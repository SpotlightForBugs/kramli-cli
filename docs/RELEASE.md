# Release setup

Releases are automated with [cargo-dist](https://github.com/axodotdev/cargo-dist). Push a semver tag (`v0.1.0`) to start the workflow.

## GitHub repositories

| Repo | Purpose |
|------|---------|
| `SpotlightForBugs/kramli-cli` | Source and GitHub Releases |
| `SpotlightForBugs/homebrew-tap` | Homebrew formulae (`brew install SpotlightForBugs/tap/kramli`) |

## Required secrets (`kramli-cli`)

### macOS code signing (cargo-dist)

| Secret | Description |
|--------|-------------|
| `APPLE_CERTIFICATE` | Base64-encoded `.p12` Developer ID Application certificate |
| `APPLE_CERTIFICATE_PASSWORD` | Password for the `.p12` file |
| `APPLE_SIGNING_IDENTITY` | e.g. `Developer ID Application: Your Name (TEAMID)` |

Export from Keychain Access, then:

```bash
base64 -i DeveloperID.p12 | pbcopy
```

### macOS notarization (custom step)

Uses an [App Store Connect API key](https://developer.apple.com/documentation/appstoreconnectapi/creating_api_keys_for_app_store_connect_api) (preferred over app-specific passwords in CI).

| Secret | Description |
|--------|-------------|
| `APPLE_NOTARY_KEY` | Base64-encoded `.p8` key file |
| `APPLE_NOTARY_KEY_ID` | Key ID from App Store Connect |
| `APPLE_NOTARY_ISSUER` | Issuer ID from App Store Connect |

### Homebrew tap

| Secret | Description |
|--------|-------------|
| `HOMEBREW_TAP_TOKEN` | GitHub PAT with `repo` scope, able to push to `homebrew-tap` |

Create the token under GitHub Settings → Developer settings → Fine-grained or classic tokens.

## Local dry run

```bash
curl --proto '=https' --tlsv1.2 -LsSf https://github.com/axodotdev/cargo-dist/releases/latest/download/cargo-dist-installer.sh | sh
dist plan
dist build --artifacts=local --target=aarch64-apple-darwin
```

## Regenerating CI

After changing `dist-workspace.toml`:

```bash
dist generate
```

Re-apply the manual blocks marked `BEGIN CUSTOM` in `.github/workflows/release.yml` (Apple env vars and notarization step).

## Security properties

- Release archives include SHA256 checksums (generated by cargo-dist).
- GitHub artifact attestations are enabled (`github-attestations = true`).
- macOS binaries are signed with hardened runtime and notarized before upload.
- Signing and notarization run only on tag releases (`publishing == true`), not on pull-request dry-runs.
- API keys stay in the OS keychain; config on disk is mode `0600` on Unix.
